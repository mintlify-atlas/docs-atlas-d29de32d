## Synopsis

```bash
gitagent skills <subcommand> [options]
```

## Description

`gitagent skills` is a set of four subcommands for working with the skills system. Skills are reusable capability modules — each is a directory containing a `SKILL.md` file with YAML frontmatter and Markdown instructions.

Skills are discovered from multiple locations:

| Priority | Path | Source |
|----------|------|--------|
| 1 | `<agent>/skills/` | Agent-local |
| 2 | `<agent>/.agents/skills/` | agentskills.io standard |
| 3 | `<agent>/.claude/skills/` | Claude Code |
| 4 | `<agent>/.github/skills/` | GitHub |
| 5 | `~/.agents/skills/` | Personal (global) |

When skills appear in multiple locations, the first match by name wins.

## skills search

Search a registry for available skills.

### Synopsis

```bash
gitagent skills search <query> [options]
```

### Options

<ParamField path="query" type="string" required>
  Search query string.
</ParamField>

<ParamField path="-p, --provider" type="string" default="skillsmp">
  Registry provider. One of `skillsmp` or `github`.
</ParamField>

<ParamField path="-d, --dir" type="string" default=".">
  Agent directory. Used to read registry configuration from `agent.yaml` when no `--provider` is specified.
</ParamField>

<ParamField path="-l, --limit" type="number" default="20">
  Maximum number of results to return.
</ParamField>

### Providers

| Provider | Description |
|----------|-------------|
| `skillsmp` | SkillsMP REST API (`https://api.skillsmp.com`) — default |
| `github` | GitHub code search |

If neither `--provider` is given nor a `registries` entry exists in `agent.yaml`, the `skillsmp` provider is used.

### Examples

```bash
gitagent skills search "code review"
gitagent skills search "pdf" --provider skillsmp --limit 5
gitagent skills search "security" -p github
```

## skills install

Install a skill from a registry or local path.

### Synopsis

```bash
gitagent skills install <name> [options]
```

### Options

<ParamField path="name" type="string" required>
  Skill name or reference. For the `github` provider, use the format `owner/repo#path/to/skill`.
</ParamField>

<ParamField path="-p, --provider" type="string" default="skillsmp">
  Registry provider. One of `skillsmp`, `github`, or `local`.
</ParamField>

<ParamField path="-g, --global" type="boolean" default="false">
  Install to the personal skills directory (`~/.agents/skills/`) instead of the agent-local `skills/` directory.
</ParamField>

<ParamField path="-d, --dir" type="string" default=".">
  Agent directory for agent-local installs (ignored when `--global` is used).
</ParamField>

### Providers

| Provider | How install works |
|----------|------------------|
| `skillsmp` | Download and extract from SkillsMP registry |
| `github` | Sparse-clone from `owner/repo#path/to/skill` |
| `local` | Copy from a local filesystem path |

### Examples

```bash
# Install from SkillsMP (default)
gitagent skills install code-review

# Install globally (shared across all agents)
gitagent skills install code-review --global

# Install from GitHub using sparse clone
gitagent skills install opencode-ai/skills#skills/pdf-reader -p github

# Install from a local path
gitagent skills install ./local-skills/my-skill -p local
```

<Tip>
  Use `--global` to install a skill once and make it available to all agents on your machine. Globally installed skills are discovered at `~/.agents/skills/`.
</Tip>

## skills list

List all skills discovered for an agent.

### Synopsis

```bash
gitagent skills list [options]
```

### Options

<ParamField path="-d, --dir" type="string" default=".">
  Agent directory to discover skills for.
</ParamField>

<ParamField path="-l, --local" type="boolean" default="false">
  Only show agent-local skills (from `<agent>/skills/`). Skips global and framework-specific locations.
</ParamField>

### Output

For each discovered skill, the command prints:

- Name
- Description
- Source path and location type
- License (if set)

### Examples

```bash
# List all discovered skills
gitagent skills list

# List only agent-local skills
gitagent skills list --local

# List skills for a specific agent
gitagent skills list -d ./examples/standard
```

## skills info

Show detailed information about a specific skill.

### Synopsis

```bash
gitagent skills info <name> [options]
```

### Options

<ParamField path="name" type="string" required>
  Name of the skill to inspect.
</ParamField>

<ParamField path="-d, --dir" type="string" default=".">
  Agent directory used for skill discovery.
</ParamField>

### Output

Displays:

- Name, description, source, directory
- License and compatibility (if set)
- Allowed tools (from `allowed-tools` frontmatter field)
- Metadata key-value pairs
- Optional sub-directories present (`scripts/`, `references/`, `assets/`, `agents/`)
- First 10 lines of the skill instructions as a preview

### Examples

```bash
gitagent skills info code-review
gitagent skills info pdf-reader -d ./my-agent
```

## SKILL.md format

Every skill must have a `SKILL.md` file with YAML frontmatter:

```markdown
---
name: code-review
description: Performs thorough code reviews with security analysis
license: MIT
compatibility: ">=0.1.0"
allowed-tools: Read Edit Grep Glob Bash
metadata:
  author: "Jane Doe"
  version: "1.0.0"
  category: "developer-tools"
---

# Instructions

Review the code for:
- Security vulnerabilities
- Performance issues
...
```

| Frontmatter field | Required | Constraints |
|-------------------|----------|-------------|
| `name` | Yes | Max 64 chars, no `--`, no leading/trailing `-` |
| `description` | Yes | Max 1024 chars |
| `license` | No | License identifier |
| `compatibility` | No | Semantic version range |
| `allowed-tools` | No | Space-delimited tool names |
| `metadata` | No | Arbitrary key-value pairs |

<Note>
  Instructions should be under ~5000 tokens (~20,000 characters). Larger instructions produce a warning during `gitagent validate`.
</Note>
