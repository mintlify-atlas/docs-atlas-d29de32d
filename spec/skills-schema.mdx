gitagent fully adopts the **Agent Skills** open standard ([agentskills.io](https://agentskills.io)) for its `skills/` directory. Any valid Agent Skills skill works in gitagent with zero modification, and gitagent skills work in Claude Code, Codex, VS Code, Cursor, Gemini CLI, and all other tools that support the standard.

## Directory structure

Each skill is a directory inside `skills/` with the skill name as the directory name:

```
skills/
└── skill-name/
    ├── SKILL.md          # Required: frontmatter + instructions
    ├── scripts/           # Optional: executable helpers
    ├── references/        # Optional: supporting docs
    ├── assets/            # Optional: templates, schemas
    ├── agents/            # Optional: agent-specific config (e.g., openai.yaml)
    └── examples/          # Optional: example inputs/outputs
```

Only `SKILL.md` is required. Everything else is optional.

## SKILL.md format

Every `SKILL.md` file has two parts: YAML frontmatter and a Markdown instructions body.

```markdown
---
name: code-review
description: Review code changes for quality, security, and best practices
license: MIT
compatibility: ">=0.1.0"
allowed-tools: lint-check complexity-analysis
metadata:
  author: gitagent-examples
  version: "1.0.0"
  category: developer-tools
---

# Code Review

## Instructions
When reviewing code:
1. Read the full diff or file provided
2. Check for security vulnerabilities (OWASP Top 10)
...
```

## Frontmatter fields

<ParamField path="name" type="string" required>
  Skill identifier. Kebab-case, max 64 characters. No consecutive hyphens, no leading or trailing hyphens. Must match the directory name.

  Pattern: `^[a-z](?:[a-z0-9]|-(?!-))*[a-z0-9]$|^[a-z]$`
</ParamField>

<ParamField path="description" type="string" required>
  What this skill does. Maximum 1024 characters. Used for skill listing and routing (metadata-only loading).
</ParamField>

<ParamField path="license" type="string">
  SPDX license identifier or `proprietary`.
</ParamField>

<ParamField path="compatibility" type="string">
  Free-text compatibility notes. Maximum 500 characters.
</ParamField>

<ParamField path="allowed-tools" type="string">
  Space-delimited list of tool names this skill may invoke. This field is experimental.

  ```yaml
  allowed-tools: lint-check complexity-analysis
  ```
</ParamField>

<ParamField path="metadata" type="object">
  Arbitrary key-value pairs. Values must be strings (per the Agent Skills standard).

  gitagent-specific extensions (`category`, `risk_tier`, `regulatory_frameworks`) live inside `metadata`, preserving full portability:

  ```yaml
  metadata:
    author: gitagent-examples
    version: "1.0.0"
    category: compliance
    risk_tier: high
    regulatory_frameworks: finra,sec,federal_reserve,cfpb
  ```
</ParamField>

## Instructions section

The body of `SKILL.md` after the frontmatter contains the skill's operational instructions. The specification recommends keeping instructions under approximately 5000 tokens (~20,000 characters) to avoid excessive context consumption.

## Progressive loading (3-tier)

Skills follow the Agent Skills standard's progressive disclosure model:

| Tier | Content | Size | When used |
|---|---|---|---|
| Metadata only | `name` + `description` | ~100 tokens | Listing and routing |
| Full skill | Frontmatter + complete instructions | ~5000 tokens max | When skill is active |
| With resources | Full skill + `scripts/`, `references/`, `assets/` | Varies | When files are needed |

## Skill discovery

Skills are discovered from multiple locations. On name collision, higher-priority sources win:

| Priority | Path | Source |
|---|---|---|
| 1 (highest) | `<agentDir>/skills/` | Agent-local |
| 2 | `<agentDir>/.agents/skills/` | agentskills.io standard |
| 3 | `<agentDir>/.claude/skills/` | Claude Code |
| 4 | `<agentDir>/.github/skills/` | GitHub |
| 5 (lowest) | `~/.agents/skills/` | Personal (global) |

## Enabling skills in agent.yaml

List skill directory names in the `skills` array in `agent.yaml`:

```yaml
skills:
  - code-review
  - security-audit
```

Each name must correspond to a directory in one of the discovery paths above.

## Installing skills from registries

```bash
# Search for a skill
gitagent skills search "code review"

# Install a skill globally
gitagent skills install code-review --global

# Install from GitHub
gitagent skills install owner/repo#path/to/skill

# List all discovered skills
gitagent skills list
```

Built-in registry providers: `skillsmp` (SkillsMP marketplace, default), `github` (GitHub code search), `local` (local filesystem).

## Validation rules

`gitagent validate` checks skills by:

1. Verifying each name in `agent.yaml skills[]` resolves to a skill directory
2. Parsing the `SKILL.md` frontmatter against the JSON schema
3. Checking that `name` matches the directory name
4. Checking that `description` is present and within the 1024-character limit
5. Warning if instructions exceed the ~5000-token recommended limit

## JSON schema

The skill frontmatter schema is at `spec/schemas/skill.schema.json`. It uses JSON Schema Draft 2020-12 and is referenced at `https://gitagent.dev/schemas/skill.schema.json`.

```json
{
  "required": ["name", "description"],
  "properties": {
    "name": { "type": "string", "maxLength": 64 },
    "description": { "type": "string", "maxLength": 1024 },
    "license": { "type": "string" },
    "compatibility": { "type": "string", "maxLength": 500 },
    "allowed-tools": { "type": "string" },
    "metadata": { "type": "object", "additionalProperties": { "type": "string" } }
  },
  "additionalProperties": false
}
```
