---
title: 'OpenAI adapter'
description: 'Generate Python code for the OpenAI Agents SDK and execute it with python3.'
---

The OpenAI adapter exports your agent as a self-contained Python module using the OpenAI Agents SDK. The runner writes the file to a temp location and executes it with `python3`.

## Export format

Running `gitagent export --format openai` produces a Python source file with three sections:

1. Module docstring with the agent name, version, and `Generated by gitagent export`
2. `from agents import Agent, Tool` import
3. Python function stubs for each tool in `tools/*.yaml`
4. `Agent()` instantiation

```bash
# Save to agent.py
gitagent export --format openai --output agent.py

# Preview
gitagent export --format openai
```

### Python function stubs

For each `tools/*.yaml` file, the adapter generates a function stub using the tool's `name`, `description`, and `input_schema`. Parameter names, types, and required/optional status come directly from the schema.

**Type mapping:**

| JSON Schema type | Python type |
|-----------------|-------------|
| `string` | `str` |
| `integer` | `int` |
| `number` | `float` |
| `boolean` | `bool` |
| `array` | `list` |
| `object` | `dict` |
| *(other)* | `str` |

Required parameters are typed without a default. Optional parameters default to `None`.

**Example tool YAML → Python stub:**

```yaml
# tools/lint-check.yaml
name: lint-check
description: Run a linter on the specified file and return findings
input_schema:
  properties:
    file_path:
      type: string
    fix:
      type: boolean
  required:
    - file_path
```

```python
def lint_check(file_path: str, fix: bool = None):
    """Run a linter on the specified file and return findings"""
    # TODO: Implement tool logic
    pass
```

<Note>
Tool names with hyphens are converted to underscores to produce valid Python identifiers (e.g., `lint-check` → `lint_check`).
</Note>

### Agent instantiation

The adapter generates an `Agent()` call with the agent's name, instructions (full system prompt), model, and tools list:

```python
code_review_agent = Agent(
    name="code-review",
    instructions="""<SOUL.md + RULES.md + skills + compliance>""",
    model="claude-sonnet-4-5-20250929",
    tools=[lint_check, complexity_analysis],
)
```

The variable name is derived from the agent's `name` field with hyphens replaced by underscores and `_agent` appended.

### System prompt contents

The `instructions` field is built from (in order):

1. `SOUL.md`
2. `RULES.md`
3. Each skill: name, description, allowed tools, and full instructions
4. `agents/openai.yaml` — if present, appended as a YAML block for MCP dependency configuration
5. Compliance constraints (fair/balanced, no misleading, PII redaction, human-in-the-loop)

## Runner behavior

<Steps>
  <Step title="Check OPENAI_API_KEY">
    The runner verifies that `OPENAI_API_KEY` is set in the environment. If not, it exits with an error and instructions.
  </Step>
  <Step title="Export Python code">
    Calls the OpenAI adapter to generate the Python module string.
  </Step>
  <Step title="Write to temp file">
    Writes the generated code to a temporary `.py` file (e.g., `/tmp/gitagent-a1b2c3d4.py`).
  </Step>
  <Step title="Execute with python3">
    Spawns `python3 <tmpfile>` with stdio inherited, running in the agent directory with the current environment.
  </Step>
  <Step title="Clean up">
    Deletes the temp file after execution, whether or not it succeeded.
  </Step>
</Steps>

```bash
export OPENAI_API_KEY="sk-..."
gitagent run -d ./my-agent -a openai
```

## Requirements

- Python 3 available as `python3`
- `openai-agents` package installed: `pip install openai-agents`
- `OPENAI_API_KEY` environment variable set

<Warning>
The generated Python stubs have `# TODO: Implement tool logic` bodies. For production use, fill in the tool implementations before executing, or extend the exported file.
</Warning>
