---
title: 'CrewAI adapter'
description: 'Export your agent as a CrewAI YAML configuration and execute it with crewai kickoff.'
---

The CrewAI adapter converts your agent definition into a CrewAI crew configuration YAML file. It parses `SOUL.md` to extract the role, goal, and backstory required by CrewAI, and maps sub-agents from your `agents/` directory into additional crew members.

## Export format

Running `gitagent export --format crewai` produces a YAML file with a top-level `agents` key.

```bash
# Save to crew.yaml
gitagent export --format crewai --output crew.yaml

# Preview
gitagent export --format crewai -d ./my-agent
```

### SOUL.md parsing

The adapter parses `SOUL.md` section headings (any `#`, `##`, or `###` heading) into named sections:

| SOUL.md section | Maps to CrewAI field | Fallback |
|----------------|---------------------|----------|
| `Core Identity` | `role` | `agent.yaml` `description` |
| `Values & Principles` | `goal` | `agent.yaml` `description` |
| *(full document, headings stripped)* | `backstory` | empty string |

- `role` is truncated to 500 characters
- `goal` is truncated to 500 characters  
- `backstory` is truncated to 2000 characters

### Skill descriptions in backstory

Skill descriptions are appended to the backstory so CrewAI is aware of the agent's capabilities:

```yaml
# appended to backstory:
Capabilities (Skills):
- code-review: Performs thorough code reviews with security analysis
- security-audit: Scans for common vulnerability patterns
```

### Agent configuration fields

| Field | Value | Notes |
|-------|-------|-------|
| `role` | From SOUL.md `Core Identity` | Truncated to 500 chars |
| `goal` | From SOUL.md `Values & Principles` | Truncated to 500 chars |
| `backstory` | SOUL.md (headings stripped) + skill descriptions | Truncated to 2000 chars |
| `verbose` | `true` | Always enabled |
| `allow_delegation` | `true` when `delegation.mode: auto` | From `agent.yaml` |

### Sub-agents

Each entry in `agent.yaml`'s `agents` map becomes an additional agent in the crew. The sub-agent's `SOUL.md` (if present) is used as its backstory.

**Example output:**

```yaml
# CrewAI configuration generated by gitagent export
# Source: code-review-agent v1.0.0

agents:
  code-review-agent:
    role: Senior code review specialist...
    goal: Ensure all code meets security and quality standards...
    backstory: |
      I approach every code review with a security-first mindset...

      Capabilities (Skills):
      - code-review: Performs thorough code reviews with security analysis
    verbose: true
    allow_delegation: false

  fact-checker:
    role: Verifies factual claims
    goal: Verifies factual claims
    backstory: ''
    verbose: true
```

## Runner behavior

<Steps>
  <Step title="Export CrewAI YAML">
    Calls the CrewAI adapter to generate the configuration string.
  </Step>
  <Step title="Write to temp file">
    Writes the YAML to a temporary file (e.g., `/tmp/gitagent-a1b2c3d4.yaml`).
  </Step>
  <Step title="Run crewai kickoff">
    Executes `crewai kickoff --config <tmpfile>` with stdio inherited, running in the agent directory.
  </Step>
  <Step title="Clean up">
    Deletes the temp file after execution.
  </Step>
</Steps>

```bash
gitagent run -d ./my-agent -a crewai
```

## Requirements

- `crewai` CLI installed:

```bash
pip install crewai
```

<Tip>
You can export the YAML first with `gitagent export --format crewai --output crew.yaml`, inspect and edit it, then run it directly with `crewai kickoff --config crew.yaml` for full control.
</Tip>
