## Your repository becomes your agent

GitAgent defines a portable, version-controlled, human-readable format for AI agent definitions. An agent is fully described by files in a git repository. Cloning the repo gives you a complete agent — ready to run on any framework.

## Why GitAgent?

Every AI framework has its own structure. There's no universal, portable way to define an agent that works across Claude Code, OpenAI, LangChain, CrewAI, and AutoGen. GitAgent fixes that.

## The standard in two files

Only two files are required to define a valid agent:

```yaml agent.yaml
spec_version: "0.1.0"
name: my-agent
version: 0.1.0
description: A helpful assistant agent
```

```markdown SOUL.md
# My Agent

## Core Identity
I am a helpful assistant that...
```

Everything else — skills, tools, workflows, compliance, memory — is optional. Add what you need, ignore the rest.

## Key features

- **10 CLI commands** — `init`, `validate`, `info`, `export`, `import`, `install`, `run`, `audit`, `skills`, `lyzr`
- **9 export adapters** — system-prompt, claude-code, openai, crewai, openclaw, nanobot, lyzr, github, git
- **Full compliance stack** — FINRA Rules 3110/4511/2210, SR 11-7, SR 23-4, Reg S-P, CFPB
- **Skills registry** — Search and install reusable capability modules from SkillsMP or GitHub
- **Git caching** — Fast repeated runs from remote repos via `~/.gitagent/cache/`

## Specification

GitAgent is an open standard. The full specification is available at [`spec/SPECIFICATION.md`](https://github.com/open-gitagent/gitagent/blob/main/spec/SPECIFICATION.md) with JSON schemas for validation at `spec/schemas/`.

<Note>
  GitAgent is published on npm as `@shreyaskapale/gitagent`. Install with `npm install -g gitagent`.
